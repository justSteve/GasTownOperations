Unified Engine

Zgents = Autonomous Claude Code agents that publish APIs and follow shared conventions

Gas Town coordinates 20-30+ agents simultaneously on software projects. Each "Zgent" is a well-formed agent that can interact with other Zgents. To scale this, we need reliable infrastructure - not manual configuration.

The Scale Challenge

Manual configuration doesn't scale

At 1-2 Agents

  • Hand-edit config files
  • Remember what changed
  • Hope nothing breaks

At 20+ Agents

  • Programmatic management
  • Full audit trail
  • Automated validation
When you have 1-2 agents, manual configuration works fine. At scale, you need automation, observability, and reliability. This is the gap CrudEngine fills.

Before CrudEngine

No Audit Trail - Who changed what? When? Why?
Manual File Manipulation - Error-prone, inconsistent
No Visibility - What are agents creating/modifying?
No Versioning - Can't roll back bad changes

Result: Cannot reliably scale beyond a handful of agents

These pain points block production deployment of Zgents. Without solving them, we're stuck at small scale.

Introducing CrudEngine

The central nervous system for Claude Code configuration

Unified CRUD Interface - Same pattern for all 7 artifact types
Full Observability - Every operation logged with context
Built-in Versioning - Before/after snapshots for all changes
Real-time Subscriptions - React to changes as they happen
CrudEngine is the lifecycle manager for Claude Code artifacts. Think of it as the database layer that all agents go through. Every operation is observed, logged, and versioned.
